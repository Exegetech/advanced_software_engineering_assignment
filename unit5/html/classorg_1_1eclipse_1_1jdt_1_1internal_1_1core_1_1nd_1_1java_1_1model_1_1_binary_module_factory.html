Read the class file from disk, circumventing the index's cache. This should only be used by callers that need to read information from the class file which aren't present in the index (such as method bodies).

Returns
the newly-created IBinaryModule or null if the given class file does not exist.

ClassFormatExceptionif the class file existed but was corrupt

Reads the given binary module. If the module can be found in the index with a fingerprint that exactly matches the file on disk, the type is read from the index. Otherwise the type is read from disk. Returns null if no such type exists. caveat modules are not yet supported in the index.
