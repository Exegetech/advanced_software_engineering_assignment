Detailed Description

This interface denotes a compilation unit, providing its name and content.

Note: This internal interface has been implemented illegally by the org.apache.jasper.glassfish bundle from Orbit, see bug 500211. Avoid changing the API or supply default methods to avoid breaking the Eclipse Help system.

◆ getContents()

char[] org.eclipse.jdt.internal.compiler.env.ICompilationUnit.getContents ( )

Answer the contents of the compilation unit.

In normal use, the contents are requested twice. Once during the initial lite parsing step, then again for the more detailed parsing step. Implementors must never return null - return an empty char[] instead, CharOperation.NO_CHAR being the candidate of choice.

Implemented in org.eclipse.jdt.internal.compiler.batch.CompilationUnit, org.eclipse.jdt.internal.core.builder.SourceFile, and org.eclipse.jdt.internal.core.CompilationUnit.
